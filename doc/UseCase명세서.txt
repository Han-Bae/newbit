제목:
	로그인
개요:
	본 유즈케이스는 시스템에 진입하기 위해서 로그인을 제공하는 기능이다.
관련액터:
	고객(사용자), 관리
기본흐름:
	1. 고객은 시스템에 로그인 화면을 요청한다. 
	2. 고객은 아이디와 암호를 입력한다.
	3. 고객은 로그인 버튼을 누른다.
	4. 시스템은 고객의 정보를 확인한다.
	5. 고객정보가 일치하면 로그인 처리 후, 메인화면으로 이동시킨다.
대체흐름:
	1-A: 회원만 사용할 수 있는 기능을 선택한 경우, 자동으로 로그인 화면으로 이동시킨다.
	2-A: 아이디와 암호는 ~~규칙을 지켜야 하며 규칙에 위배된 내용이 
			입력되면 해당 입력란을 초기화 시킨다.
	5-A: 데이터베이스에 회원정보가 없다면, 다시 로그인 화면으로 이동시킨다.
예외흐름:
	5-A: 데이터베이스 접속이 원활하지 않은 경우 에러 페이지로 이동시킨다.
확장점:
	1. 고객이 아닌경우 회원가입을 할 수 있도록 버튼을 제공한다.
	2. 원가입을 하게되면 자동으로 로그인 된 것으로 처리한다.
	3. 아이디를 분실한 고객을 위해서 아이디 찾기 기능을 위한 버튼을 제공한다.
	4. 암호를 분실한 고객을 위해서 암호찾기 기능을 위한 버튼을 제공한다.
	
제목:
	회원가입
개요:
	본 유즈케이스는 비회원이 회원가입을 하기위한 회원가입을 제공하는 기능.
기본흐름:
	1. 비회원은 시스템에 회원가입을 요청한다.
	2. 비회원은 개인정보를 입력한다._id, pw, name, mail, tel, gen, avt
	3. 비회원은 아이디중복확인 버튼을 누른다.
	4. 시스템은 아이디의 중복여부를 확인한다.
	5. 비회원은 이메일중복확인 버튼을 누른다.
	6. 시스템은 이메일의 중복여부를 확인한다.
	7. 비회원은 회원가입 버튼을 누른다.
	8. 시스템은 입력된 회원정보의 유효성 검사를 마친 후 회원가입 요청을 한다.
	9. 시스템은 비회원의 정보를 Database에 입력한다.
	10. 입력이 완료되면 로그인 후 메인화면을 보여준다.
대체흐름:
	3-A: DB에 중복된 아이디가 있다면, 안내와 함께 입력란을 초기화 시킨다.
	5-A: DB에 중복된 이메일이 있다면, 안내와 함께 입력란을 초기화 시킨다.
	7-A: 누락된 입력란이 있다면, 안내와 함께 입력란으로 돌려보낸다.
	8-A: 유효하지 않은 데이터가 있다면 알림과 함께 입력란을 초기화 하고 입력란으로 돌려보낸다.
예외흐름:
	3-A: 데이터베이스 접속이 원활하지 않은 경우, '잠시후 시도하라'는 안내 메세지를 보낸다.
	5-A: 데이터베이스 접속이 원활하지 않은 경우, '잠시후 시도하라'는 안내 메세지를 보낸다.
	3-A: 데이터베이스 접속이 원활하지 않은 경우, 회원정보 입력 페이지로 돌려보낸다.
